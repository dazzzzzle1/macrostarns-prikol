# Как выложить сайт бесплатно

Проект: статика + PHP (`send.php`, `admin.php`, SQLite в папке `data/`).

## Вариант A — Render.com (Docker, проще всего из GitHub)

1. Зарегистрируйтесь на [render.com](https://render.com) и подключите GitHub.
2. Создайте **новый репозиторий** на GitHub, залейте **всю папку проекта** (включая `assets/`, если есть).
3. В Render: **New → Web Service**, выберите репозиторий.
4. Укажите **Docker** (Render сам увидит `Dockerfile` или используйте `render.yaml`).
5. В разделе **Environment** добавьте переменную **`ADMIN_PASS`** — свой сложный пароль (не оставляйте `admin123` в интернете).
6. После деплоя откройте выданный URL вида `https://ваш-сервис.onrender.com`.

**Минус бесплатного тарифа:** сервис «засыпает» без посещений, первый заход может быть медленным. **Файл SQLite иногда теряется** при перезапуске/новом деплое — периодически скачивайте заявки из админки (**Скачать CSV**).

## Вариант B — бесплатный PHP-хостинг (FTP, база обычно сохраняется)

Подойдут сервисы вроде **InfinityFree**, **ProFreeHost**, **000webhost** (набор меняется, ищите «free PHP hosting»).

1. Зарегистрируйтесь и создайте сайт, получите **FTP** и адрес вида `https://ваш-поддомен.хост.com`.
2. Загрузите **все файлы** в каталог `public_html` (или `htdocs`, как укажет хостинг).
3. В панели или по FTP создайте папку **`data`**, выставьте права **на запись** (часто `755` или `775`; если не создаётся база — попробуйте `777` только для `data`, это слабее с точки зрения безопасности).
4. Убедитесь, что на хостинге включены **PHP** и расширения **pdo_sqlite** (и по возможности **mbstring**). Это смотрят в панели или в `phpinfo()`.
5. Пароль админки задайте переменной **`ADMIN_PASS`** в панели (если есть), иначе смените в коде только для теста — лучше не хранить пароль в открытом виде в репозитории.

Добавлен файл **`data/.htaccess`** с запретом доступа к файлам в этой папке (работает на Apache).

## Вариант C — свой VPS с Docker

На любом дешёвом VPS:

```bash
git clone <ваш-репозиторий> && cd papkf
docker build -t macrotrans .
docker run -d -p 8080:80 -e ADMIN_PASS='сложный-пароль' -v macrotrans-data:/var/www/html/data --name macrotrans macrotrans
```

Том `-v` сохраняет SQLite между перезапусками.

## Обязательно после публикации

- Сменить **`ADMIN_PASS`** в настройках хостинга (уже поддерживается в `admin.php` / `admin_auth.php`).
- Не выкладывать в открытый Git реальный пароль и личные данные клиентов.
